ASP.NET Core MVC CRUD (Database First) BankSystem ------------------------------------------------------------------------------------------------------------------------------------------------------------ 1. Create Project - Open Visual Studio 2022 - Create New Project - Choose ASP.NET Core Web App (Model-View-Controller) - Project Name: BankSystem - Framework: .NET 8 (or lab version) - Authentication: None - Check Configure for HTTPS - Create ---------------------------------------------------------------------------------------------------------------------------------------------------------- 2. Install NuGet Packages - Microsoft.EntityFrameworkCore.SqlServer - Microsoft.EntityFrameworkCore.Design - Microsoft.EntityFrameworkCore.Tools - Microsoft.VisualStudio.Web.CodeGeneration.Design (recommended) ------------------------------------------------------------------------------------------------------------------------------------------------------ 3. Create Database - View -> SQL Server Object Explorer - (localdb)-> Databases -> Add New Database - Name: BankDB --------------------------------------------------------------------------------------------------------------------------------------------------------------- 4. Create Table - Tables -> Add New Table Columns: Id int (Primary Key, Identity=True) name nvarchar(50) accountno int address nvarchar(100) - Update -> Table Name: Account -> Update Database - (Optional) Insert sample data ------------------------------------------------------------------------------------------------------------------------------------------------------ 5. Scaffold Database First Tools -> NuGet Package Manager -> Package Manager Console Scaffold-DbContext “Data Source=(localdb);Initial Catalog=BankDB;Integrated Security=True;Connect Timeout=30;Encrypt=True;Trust Server Certificate=False;Application Intent=ReadWrite;Multi Subnet Failover=False;Command Timeout=30” Microsoft.EntityFrameworkCore.SqlServer -OutputDir Models Generated: - Models/Account.cs - Models/BankDbContext.cs Delete only the #warning line in BankDbContext.cs. --------------------------------------------------------------------------------------------------------------------------------------------------- Step 6 - appsettings.json Open appsettings.json and replace it with: { "Logging": { "LogLevel": { "Default": "Information", "Microsoft.AspNetCore": "Warning" } }, "AllowedHosts": "*", "ConnectionStrings": { "DefaultConnection": "Data Source=(localdb)\\MSSQLLocalDB;Initial Catalog=BankDB;Integrated Security=True;Connect Timeout=30;Encrypt=True;Trust Server Certificate=False;Application Intent=ReadWrite;Multi Subnet Failover=False;Command Timeout=30" } } Save the file. ------------------------------------------------------------------------ Step 7 - Program.cs Replace Program.cs with: using Microsoft.EntityFrameworkCore; using BankSystem.Models; var builder = WebApplication.CreateBuilder(args); string connection = builder.Configuration.GetConnectionString("DefaultConnection"); builder.Services.AddDbContext(options => { options.UseSqlServer(connection); }); builder.Services.AddControllersWithViews(); var app = builder.Build(); if (!app.Environment.IsDevelopment()) { app.UseExceptionHandler("/Home/Error"); app.UseHsts(); } app.UseHttpsRedirection(); app.UseRouting(); //app.UseAuthorization(); app.MapStaticAssets(); app.MapControllerRoute( name: "default", pattern: "{controller=Home}/{action=Index}/{id?}") .WithStaticAssets(); app.Run(); Build the solution. ------------------------------------------------------------------------ Step 8 Build → Build Solution No errors should remain. ------------------------------------------------------------------------ Step 9 - Generate MVC CRUD Right Click Controllers → Add → New Scaffolded Item → MVC Controller with views, using Entity Framework Select: Model class: Account Data context class: BankDbContext Controller name: AccountsController Click Add. Generated automatically: Controllers - AccountsController.cs Views - Account - Index.cshtml - Create.cshtml - Edit.cshtml - Delete.cshtml - Details.cshtml ------------------------------------------------------------------------ Step 10 - Rename Views Folder Visual Studio usually creates Views/Account Rename it to Views/Accounts because your controller is AccountsController.cs The controller name and View folder name must match. Final folder: Views Accounts Create.cshtml Delete.cshtml Details.cshtml Edit.cshtml Index.cshtml ------------------------------------------------------------------------ Step 11 - Navigation Menu Open Views → Shared → _Layout.cshtml Add inside the navbar:
  • BankSystem
    • Save. ------------------------------------------------------------------------ Step 12 - Run Press Ctrl+F5 Open https://localhost:PORT/Accounts ------------------------------------------------------------------------ Generated CRUD Index - Displays all records. Create - Adds a new record. Details - Finds record by Id. Edit - Updates record by Id. Delete - Deletes record by Id. Automatically included: - ModelState.IsValid - NotFound() - DbUpdateConcurrencyException Keep these files: - HomeController.cs - ErrorViewModel.cs - Views/Home - Views/Shared Do not delete them.
